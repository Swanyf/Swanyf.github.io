基本命令

v-clock

能够解决页面加载时出现原生表达式&#123; &#125;的问题

v-text v-html

会覆盖原本的标签值，v-html能解析带有标签的内容

v-bind

简写:，可以将标签属性转为vue属性，多用来绑定值

v-on

简写@，可以自定义绑定事件，也可以绑定html事件

事件描述符

  • .stop，阻止冒泡事件
  • .prevent，阻止默认行为
  • .capture，也就是冒泡反过来由外到内
  • .self，只有是自己触发的自己才会执行,如果接受到内部的冒泡事件传递信号触发,会忽略掉这个信号
  • .once，将事件设置为只执行一次,如.click.prevent.once代表只阻止事件的默认行为一次,当第二次触发的时候事件本身的行为会执行

v-model

双向动态数据绑定，只能作用于表单元素

v-for

遍历数据

v-if

每次都会重新删除或创建dom元素

v-show

每次只是修改元素样式为display: none

filter过滤器

####全局过滤器
使用管道符|调用过滤器，可以同时使用多个过滤器层级过滤数据

  • 定义，Vue.filter('过滤器名称', function(data, ...) { })
  • 调用，xxx | '过滤器名称'

vuex

Vue配套的公共数据管理工具，可以把一些共享的数据保存到vuex中，方便整个程序中的任何组件直接获取或修改公共数据，实现了各组件数据的共享与调用

使用

  • 安装cnpm install vuex --save

  • 引入

  • 入口函数

  • 通过this.$store.state.xxx来获取数据

  • 通过this.$store.commit('myMethod', {id: 1, age: 20})调用 mutations 中的方法，方法的参数列表只支持俩个参数：state状态、通过commit传递的参数，若要传递多个参数可以通过对象的方式传递

  • store中的state数据在对外提供时，推荐使用getters属性去包装，通过this.$store.getters.myCount调用

  • getters中的方法和组件中过滤器比较类似，都是没有修改原数据，在原数据的基础上进行包装提供给调用者。当state中的数据发生改变且getters引用的该数据，就会立即出发getters的重新请求值
