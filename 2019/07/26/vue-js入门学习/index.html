v-bind

简写:，可以将标签属性转为vue属性，多用来绑定值

v-on

简写@，可以自定义绑定事件，也可以绑定html事件

事件描述符

  • .stop，阻止冒泡事件
  • .prevent，阻止默认行为
  • .capture，也就是冒泡反过来由外到内
  • .self，只有是自己触发的自己才会执行,如果接受到内部的冒泡事件传递信号触发,会忽略掉这个信号
  • .once，将事件设置为只执行一次,如.click.prevent.once代表只阻止事件的默认行为一次,当第二次触发的时候事件本身的行为会执行

v-model

双向动态数据绑定，只能作用于表单元素

v-for

遍历数据

v-if

每次都会重新删除或创建dom元素

v-show

每次只是修改元素样式为display: none

vue组件

定义

为了拆分 vue 实例中的代码量，能够以不同的组件去划分不同的功能模块。需要什么样的功能就调用什么样的组件

组件化和模块化的区别

  • 组件化： 从 UI 界面的角度进行划分，前端的组件化，方便 UI 组件的重用
  • 模块化： 从代码逻辑的角度进行划分，方便代码的分层开发，保证每个功能模块的单一性

私有组件

父组件向子组件传值

  • 组件中所有的props数据，都是从父组件绑定的属性传递给子组件的
  • 子组件中默认无法访问父组件传递的数据和方法
  • props中的数据为只读，不能重新赋值
  • data中的数据可读可写

router路由

介绍

  • 后端路由 -> 对于普通的网站，所有的超链接都是url地址，所有的url地址都对应服务器上的对应资源
  • 前端路由 -> 对于单页面应用程序来说，主要通过url中的hash(#号)来实现不同页面之间的跳转，
  • hash 的特点 -> http请求中不会包含hash的相关内容，所以单页面程序中的跳转主要用于hash实现单页面程序，通过hash来改变切换页面的方式，成为前端路由

路由执行流程

    1. 点击链接改变url地址，url地址触发路由的监听事件
    1. 然后进行路由规则匹配
    1. 如果能匹配到，就展示对应的组件
    1. 最后由<router-view>标签显示在页面上

语法

new 路由对象时，可以为构造函数传递一个配置对象

  • <router-view></router-view> -> 在前端渲染被控制的组件
  • <router-link to="/xxx"></router-link> -> 在前端被渲染成标签

vuex

Vue配套的公共数据管理工具，可以把一些共享的数据保存到vuex中，方便整个程序中的任何组件直接获取或修改公共数据，实现了各组件数据的共享与调用

使用

  • 安装cnpm install vuex --save

  • 引入

  • 入口函数

  • 通过this.$store.state.xxx来获取数据

  • 通过this.$store.commit('myMethod', {id: 1, age: 20})调用 mutations 中的方法，方法的参数列表只支持俩个参数：state状态、通过commit传递的参数，若要传递多个参数可以通过对象的方式传递

  • store中的state数据在对外提供时，推荐使用getters属性去包装，通过this.$store.getters.myCount调用

  • getters中的方法和组件中过滤器比较类似，都是没有修改原数据，在原数据的基础上进行包装提供给调用者。当state中的数据发生改变且getters引用的该数据，就会立即出发getters的重新请求值
